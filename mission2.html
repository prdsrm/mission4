Mission 2: Exploitation BDD KDou avec une interface low code

Contexte

La deuxième mission visait à exploiter la base de données de KDOU en utilisant une interface low code. L'objectif était d'extraire des indicateurs pertinents pour l'analyse de l'activité de l'entreprise.

Présentation des indicateurs

Indicateur 1: Analyse des ventes par catégorie de produits

Énoncé et interprétation: Cet indicateur vise à déterminer la répartition des ventes par catégorie de produits, permettant d'identifier les catégories les plus performantes.

Résultat: [Insérer ici un tableau ou une capture d'écran du résultat]

Code SQL généré:


SELECT Categorie.NomCateg, SUM(DetailCommande.QteCom * DetailCommande.PrixUnitCli) AS Ventes
FROM Categorie
INNER JOIN Produit ON Categorie.CodeCateg = Produit.CodeCateg
INNER JOIN DetailCommande ON Produit.RefProd = DetailCommande.RefProd
GROUP BY Categorie.NomCateg
ORDER BY Ventes DESC;

Indicateur 2: Analyse des fournisseurs les plus actifs

Énoncé et interprétation: Cet indicateur permet d'identifier les fournisseurs dont les produits sont les plus commandés, aidant à optimiser la gestion des approvisionnements.

Résultat: [Insérer ici un tableau ou une capture d'écran du résultat]

Code SQL généré:


SELECT Fournisseur.Societe, COUNT(DISTINCT DetailCommande.RefProd) AS NbProduitsCommandes
FROM Fournisseur
INNER JOIN Produit ON Fournisseur.NoFour = Produit.NoFour
INNER JOIN DetailCommande ON Produit.RefProd = DetailCommande.RefProd
GROUP BY Fournisseur.Societe
ORDER BY NbProduitsCommandes DESC;

Conclusion de la Mission 2

L'exploitation de la base de données KDOU à l'aide d'une interface low code a permis de mettre en lumière plusieurs aspects importants de l'activité de l'entreprise. Les indicateurs analysés fournissent des informations précieuses sur les performances des catégories de produits, l'activité des fournisseurs, et d'autres aspects clés qui pourront guider les décisions stratégiques futures de KDOU.
